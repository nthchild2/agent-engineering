This folder is a temporary workspace for developing a manifesto on agentic orchestration for software development.
Capture source-backed questions, early answers, and design constraints before drafting manifesto postulates.
1. Gather primary sources from model vendors, benchmark authors, and research groups.
2. Extract durable claims about capabilities, limitations, and operating constraints.
3. Separate descriptive claims (what models do) from normative claims (how we should use them).
4. Turn repeated patterns into manifesto postulates.
5. Derive tool requirements from those postulates.
- `questions.md`: research questions worth answering before drafting the manifesto.
- `sources.md`: curated primary sources with brief notes on why each matters.
- Capability: what generative models reliably do well.
- Limitation: where they fail, drift, hallucinate, or need verification.
- Parameter: what governs behavior in practice.
- Orchestration: how multiple agents and humans should coordinate.
- Software development: what changes when the domain is code, tests, repos, and release workflows.
