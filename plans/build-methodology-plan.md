## Plan: Build Agent-Engineering Methodology

Develop a manifesto on agentic orchestration first, grounded in source-backed research, then derive tools, workflows, and evaluations for software development. Prioritize small, legible changes with artifact-centered workflows, separating descriptive claims from normative ones, and validating via experiments or benchmarks.

### Steps
1. Expand [research/sources.md](research/sources.md) with primary sources, extracting descriptive truths, constraints, patterns, and implications.
2. Refine [research/questions.md](research/questions.md) backlog, testing provisional hypotheses against sources for manifesto postulates.
3. Draft manifesto postulates in new [MANIFESTO.md](MANIFESTO.md), linking to sources and separating descriptive/normative claims.
4. Validate postulates via targeted experiments in [research/](research/), using benchmarks like SWE-bench for software dev relevance.
5. Implement initial tools from postulates, starting with simple, inspectable scripts in new [tools/](tools/) folder.
6. Design workflows and evaluations around artifacts, updating [README.md](README.md) and [AGENTS.md](AGENTS.md) for handoffs.

### Further Considerations
1. How to measure validation success: benchmarks vs. real repo tasks?
2. Scope tool development: start with one postulate or multi-agent pattern?
3. Provisional ideas: label as such in docs until validated.
